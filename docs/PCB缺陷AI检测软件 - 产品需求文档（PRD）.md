# PCB 缺陷 AI 检测软件 - 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品定位



* **名称**：PCB-AI Inspector（中文：PCB 智能检测大师）

* **核心价值**：本地离线运行，基于 YOLO11 AI 算法，快速识别 PCB 板 10 大类常见缺陷，自动生成质检报告，替代人工肉眼检测，降低中小 PCB 厂、研发工作室的质检成本。

* **目标用户**：PCB 打样厂、SMT 贴片厂、电子研发工作室、高校实验室、维修检测部门（无专业 AOI 设备、预算有限、需快速自检）

* **核心优势**：本地离线、零配置、傻瓜操作、检测速度快、支持私有化部署、价格亲民（1280-3980 元 / 永久授权）

### 1.2 产品形态



* 运行环境：Windows 10/11（64 位），支持 CPU/GPU 加速（GPU 可选，CPU 即可实时运行）

* 软件形态：桌面客户端（.exe 文件，一键安装 / 绿色免安装版）

* 网络要求：全程离线，无需联网，数据不上传云端

## 二、核心功能需求

### 2.1 基础功能模块

#### 2.1.1 图片导入与管理



| 功能点  | 详细需求                                               |
| ---- | -------------------------------------------------- |
| 导入方式 | 支持 3 种导入：①拖拽图片到软件界面；②点击 “导入” 选择本地文件；③批量导入文件夹（支持多选） |
| 支持格式 | 主流图片格式：JPG、PNG、BMP、TIFF（工业相机 / 扫描仪常用格式）            |
| 图片预览 | 导入后自动生成缩略图，支持放大 / 缩小 / 旋转 / 平移查看原图细节               |
| 批量处理 | 支持多图片批量排队检测，自动按顺序处理                                |

#### 2.1.2 AI 缺陷检测（核心模块）



| 功能点    | 详细需求                                                                                                                                                                       |
| ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 检测缺陷类型 | 6 大类标准缺陷（覆盖行业 80% 以上场景），**精确定义如下**：<br><br>1. **短路（short）**：两条或多条相邻导线/焊盘之间距离 < 最小间距阈值（默认 0.2mm，即 8mil），或导线与焊盘异常连接<br>2. **开路（open_circuit）**：导线或焊盘存在断裂，断裂长度 ≥ 最小线宽，且断点两侧铜皮未连接<br>3. **缺孔（missing_hole）**：设计应有孔的位置（根据 PCB 设计文件或模板对比），实际缺失孔径；排除 < 0.3mm 的微小孔（工艺限制）<br>4. **毛刺（spur）**：导线边缘突出部分，宽度 0.1-2mm，长度 ≥ 宽度的 2 倍；排除 < 0.1mm 的噪点<br>5. **鼠咬（mouse_bite）**：导线或铜皮边缘存在缺口，缺口宽度 0.2-1mm，深度 ≥ 最小线宽的 30%<br>6. **多余铜皮（spurious_copper）**：设计图上不存在的铜皮区域，面积 > 1mm²，与最近导线/焊盘距离 > 0.15mm |
| 检测触发   | 一键点击 “开始检测”，支持单图检测 / 批量检测切换                                                                                                                                                |
| 检测速度   | CPU（i5 及以上）：单图检测≤300ms；GPU（NVIDIA 1650 及以上）：单图检测≤100ms                                                                                                                     |
| 结果可视化  | 1. 缺陷区域用彩色框标注（严重缺陷红框，次要缺陷黄框）. 框内显示缺陷类型 + 置信度（如 “短路 - 92%”）. 支持 “原图 / 标注图” 切换查看 > 4. 缺陷列表：右侧显示所有缺陷的类型、坐标（X/Y）、尺寸（宽 / 高）、置信度                                                 |
| 检测参数可调 | 支持自定义置信度阈值（默认 0.5，可在 0.3-0.9 之间调整，阈值越低检出率越高，误报率可能上升）                                                                                                                       |

#### 2.1.3 质检报告导出



| 功能点  | 详细需求                                                                                                                                |
| ---- | ----------------------------------------------------------------------------------------------------------------------------------- |
| 报告格式 | 支持 2 种格式：①PDF（正式质检报告，可打印）；②Excel（缺陷统计台账，便于数据分析）                                                                                     |
| 报告内容 | 必含字段：1. 基础信息：检测时间、文件名、图片尺寸、检测设备（CPU/GPU 型号）. 缺陷统计：总缺陷数、各类型缺陷数、良率判断（无缺陷 = 良，有缺陷 = 不良）>3. 缺陷详情：每个缺陷的类型、置信度、坐标、尺寸、缺陷截图。附件：原图 + 标注图嵌入报告 |
| 导出设置 | 支持自定义报告保存路径、文件名（默认按 “PCB 检测\_日期\_时间” 命名）、是否嵌入原图                                                                                     |
| 批量导出 | 批量检测完成后，支持一键导出所有图片的报告（按文件夹分类存储）                                                                                                     |

#### 2.1.4 数据本地管理


| 功能点    | 详细需求                                               |
| ------ | -------------------------------------------------- |
| 检测记录保存 | 自动保存检测历史（含原图、标注图、报告路径），支持按日期 / 文件名搜索               |
| 数据存储   | 所有数据（图片、报告、模型）均存储在本地，支持自定义存储路径，默认路径：C:\PCB-AI-Data |
| 记录清理   | 支持手动删除检测历史（单条 / 批量删除），防止占用过多磁盘空间                   |

#### 2.1.5 关键 UX 功能（MVP 必备）


| 功能点    | 详细需求                                               |
| ------ | -------------------------------------------------- |
| 撤销/重做 | 支持撤销最近 10 步操作（如删除图片、清除标注等），避免误操作无法回退             |
| 检测暂停/取消 | 批量检测时支持暂停（继续）和取消（终止）操作，暂停后可恢复检测                    |
| 缺陷筛选   | 支持按缺陷类型（多选）、置信度范围筛选缺陷列表，便于快速定位问题                   |
| 缺陷截图导出 | 支持导出单个缺陷的截图（PNG 格式），便于问题沟通和存档                         |
| 标注框编辑 | 支持手动调整检测框位置/大小，支持删除误检框、补充漏检框（编辑后不覆盖原模型结果）        |
| 快捷键支持 | 常用操作快捷键：Ctrl+I（导入）、Space（开始检测）、Ctrl+E（导出）、+/-（放大缩小）、Esc（取消） |
| 键盘导航   | 图片列表支持上下键切换，缺陷列表支持 Tab 切换                               |

### 2.2 进阶功能（可选，MVP 版本可暂不开发，后续迭代）



* 模型自定义训练：支持用户导入自有 PCB 数据集，微调 YOLO11 模型（适配特殊板型 / 缺陷）

* 缺陷标注工具：支持手动添加 / 删除 / 修改缺陷标注，导出标注文件（VOC/YOLO 格式）用于模型训练

* 多语言支持：中文 / 英文界面切换（适配外贸型客户）

* 快捷键操作：支持自定义检测 / 导出 / 放大等操作的快捷键，提升操作效率

## 三、界面设计需求

### 3.1 整体风格



* 简洁、专业、工业风，避免冗余设计

* 主色调：深蓝色（#1E3A8A）+ 白色（#FFFFFF）+ 红色（#DC2626，用于缺陷标注）

* 字体：默认微软雅黑，清晰易读，支持字体大小调整

### 3.2 界面布局（核心 3 个页面）

#### 3.2.1 主界面（默认打开页面）



* 顶部导航栏：产品 Logo + 功能按钮（导入图片、开始检测、导出报告、历史记录、设置）

* 左侧：图片列表区（缩略图形式，显示已导入图片，标注 “未检测 / 已检测 / 有缺陷” 状态）

* 中间：图片预览区（显示当前选中图片的原图 / 标注图，支持鼠标操作放大缩小）

* 右侧：缺陷详情区（检测后显示缺陷列表，未检测时显示 “点击开始检测” 提示）

* 底部：状态栏（显示当前状态：就绪 / 检测中 / 检测完成、已检测图片数 / 总图片数、当前 CPU/GPU 占用率）

#### 3.2.2 历史记录页面



* 顶部：搜索框（按日期 / 文件名搜索）+ 清理按钮（批量删除）

* 中间：历史记录列表（含序号、文件名、检测时间、缺陷数、报告路径、操作按钮 “查看报告 / 重新检测 / 删除”）

* 底部：分页控件（支持分页查看，默认每页显示 20 条）

#### 3.2.3 设置页面



* 基础设置：存储路径（自定义）、默认导出格式（PDF/Excel/ 两者都选）、界面语言（中文 / 英文）

* 检测设置：置信度阈值（滑块调整）、是否启用 GPU 加速（默认自动检测 GPU，无则用 CPU）

* 关于：软件版本、授权信息、更新检查（仅本地版本检查，不联网）、帮助文档（本地 HTML）

### 3.3 交互设计



* 导入图片：拖拽后显示 “导入成功” 弹窗，自动刷新图片列表

* 检测过程：显示进度条（单图检测显示百分比，批量检测显示已完成数 / 总数）

* 缺陷标注：鼠标 hover 到缺陷框时，高亮显示该缺陷在列表中的对应条目

* 导出报告：导出完成后显示 “报告已保存至 XXX 路径” 弹窗，支持点击路径直接打开文件夹

* 错误提示：图片格式不支持、文件损坏、权限不足时，显示明确的错误提示（如 “请导入 JPG/PNG/BMP/TIFF 格式图片”）

## 四、技术要求

### 4.1 开发语言与框架



* 主语言：Python 3.11.9（Anaconda 环境，推荐使用 conda create -n pcb-ai python=3.11.9）

* AI 核心：Ultralytics YOLO11（官方开源库，支持训练 / 推理 / 导出）

* 界面框架：PyQt6（推荐，工业软件常用，界面美观、交互流畅）或 Flet（备选，开发速度更快，轻量化）

* 图像处理：OpenCV-Python（图像预处理、标注绘制）、Pillow（图片格式解析）

* 报告生成：ReportLab（PDF 生成）、openpyxl（Excel 生成）

* 打包工具：Nuitka（推荐，编译为 exe，运行速度快、防破解）或 PyInstaller（备选，配置简单）

### 4.2 运行环境要求



* 最低配置（CPU 版）：


  * 处理器：Intel i5 8 代及以上 / AMD Ryzen 5 3 代及以上

  * 内存：8GB RAM

  * 硬盘：1GB 可用空间（含软件 + 基础模型）

  * 系统：Windows 10 64 位及以上

* 推荐配置（GPU 加速版）：


  * 显卡：NVIDIA GTX 1650 及以上（支持 CUDA 11.8+）

  * 内存：16GB RAM

  * 其他：同最低配置

### 4.3 模型要求

* 基础模型：基于 PKU PCB 公开数据集训练的 YOLO11-S 模型

* **各缺陷类型性能指标（置信度阈值 = 0.5 时）**：

| 缺陷类型 | mAP@50 | mAP@50:95 | Recall | Precision | 最小检出尺寸 |
| ---- | ------ | --------- | ------ | --------- | ---------- |
| 短路（short） | ≥0.88 | ≥0.72 | ≥0.87 | ≥0.90 | 32×32 px |
| 开路（open_circuit） | ≥0.85 | ≥0.68 | ≥0.84 | ≥0.88 | 24×24 px |
| 缺孔（missing_hole） | ≥0.90 | ≥0.75 | ≥0.89 | ≥0.92 | 16×16 px |
| 毛刺（spur） | ≥0.82 | ≥0.65 | ≥0.80 | ≥0.85 | 48×16 px |
| 鼠咬（mouse_bite） | ≥0.80 | ≥0.62 | ≥0.78 | ≥0.83 | 40×40 px |
| 多余铜皮（spurious_copper） | ≥0.87 | ≥0.70 | ≥0.86 | ≥0.89 | 48×48 px |
| **整体（6 类平均）** | **≥0.85** | **≥0.69** | **≥0.84** | **≥0.88** | - |

* **误报率控制**：在默认阈值 0.5 下，整体误报率（False Positive Rate）≤ 8%
* **漏检率控制**：在默认阈值 0.5 下，整体漏检率（Miss Rate）≤ 12%
* **适用板型**：单面板、双面板、多层板（≤ 12 层）均适用
* **输入图片规格**：
  - 支持分辨率：640×640 ~ 8192×8192 px
  - 支持 DPI：72 ~ 1200 DPI
  - 图片过大时自动分块推理（分块大小 2048×2048 px，重叠 128 px），拼接后输出
* 模型部署：导出为 ONNX 格式，本地离线推理，无需额外下载模型
* 模型大小：≤50MB（保证软件安装包体积小巧，下载 / 安装快速）
* **数据集使用声明**：PKU PCB 数据集仅用于学术研究，商业使用需确认 License 或自行采集数据

### 4.4 安全与兼容性


* 安全：无恶意代码、不收集用户数据、不访问网络、不修改系统配置
* **激活机制**：
  - 授权方式：离线激活码（硬件指纹 + 激活码 + 签名验证）
  - 硬件指纹：基于 CPU 序列号 + 主板序列号 + 硬盘序列号的 Hash 值
  - 激活码格式：XXXX-XXXX-XXXX-XXXX（16位，分4组）
  - 激活验证：激活时验证激活码签名与硬件指纹匹配，无需联网
  - 重装支持：同一机器支持无限次重装（基于硬件指纹不变）
  - 更换硬件：支持最多 2 次硬件变更申请（需联系客服）
* 兼容性：支持 Windows 10/11（64 位，家庭版 / 专业版 / 企业版），**不支持 32 位系统（PyQt6 限制）**
* 稳定性：连续检测 1000 张图片无崩溃，无内存泄漏

### 4.5 日志与运维支持

* **日志系统**：
  - 日志级别：DEBUG / INFO / WARNING / ERROR / CRITICAL
  - 日志内容：启动时间、硬件信息、检测任务（图片路径、数量、耗时、结果摘要）、异常堆栈
  - 日志文件位置：`C:\PCB-AI-Data\logs\app_{日期}.log`
  - 日志保留：默认保留 30 天，支持用户在设置中调整
  - 日志大小限制：单文件最大 10MB，超限自动轮转（保留最近 5 个文件）

* **诊断工具**：
  - 支持导出诊断包（含日志、硬件信息、软件版本、崩溃转储）
  - 用户可在「帮助」→「提交反馈」中一键生成诊断包并复制到剪贴板
  - 诊断包不包含任何用户图片或检测数据，仅包含日志和技术信息

* **崩溃恢复**：
  - 软件崩溃后重启时，自动检测上次未完成的任务，提示用户是否恢复
  - 自动备份检测进度，每检测 10 张图片保存一次中间状态

## 五、用户流程

### 5.1 核心使用流程（5 步闭环）



1. 启动软件：双击 exe 文件，无联网验证，直接进入主界面

2. 导入图片：拖拽本地 PCB 图片或选择文件夹批量导入

3. 开始检测：点击 “开始检测”，等待进度条完成

4. 查看结果：在预览区查看标注好的缺陷，在右侧列表查看缺陷详情

5. 导出报告：选择 PDF/Excel 格式，一键导出质检报告

### 5.2 异常流程处理

| 异常场景 | 处理方式 | 用户提示 | 是否阻断批量 |
| -------- | -------- | -------- | ------------ |
| 图片无法打开 | 跳过该图片，记录错误日志 | “第 N 张图片无法打开（文件损坏或格式不支持），已跳过” | 否（批量模式继续） |
| 图片格式不支持 | 不导入，显示错误 | “请导入 JPG/PNG/BMP/TIFF 格式图片” | 是（单张导入） |
| 检测无缺陷 | 正常显示结果 | “未检测到缺陷，该 PCB 板判定为良品” | 不适用 |
| 置信度低（<0.5） | 标注但以黄色显示 | “检测到低置信度缺陷（<0.5），建议降低阈值重新检测” | 否 |
| 批量检测中取消 | 停止检测，保存已检测结果 | “已停止检测，已检测 N 张图片已保存” | 不适用 |
| 批量检测中暂停 | 暂停处理，当前图片完成后停止 | “检测已暂停，点击继续恢复” | 不适用 |
| 磁盘空间不足 | 停止写入，提示清理 | “磁盘空间不足（剩余 < 100MB），请清理后重试” | 是 |
| 内存不足（OOM） | 停止检测，保存状态 | “内存不足，请关闭其他程序后重试” | 是 |
| 授权过期/无效 | 限制检测功能 | “授权无效，请重新激活” | 是（仅预览可用） |
| 导出报告失败 | 提示错误，保留数据 | “报告导出失败（原因：XXX），请检查路径权限” | 否（可重试） |
| 图片尺寸超出限制（> 8192px） | 提示并建议缩放 | “图片尺寸超出限制（最大 8192×8192 px），请先缩放后重试” | 是 |
