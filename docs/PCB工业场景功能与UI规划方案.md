# PCB 缺陷 AI 检测系统 - 工业场景功能与 UI 规划方案

## 一、工业场景需求分析

### 1.1 核心用户角色

工业场景下存在三种典型用户角色，其需求差异显著：

| 角色 | 使用场景 | 核心需求 | UI 偏好 |
|------|----------|----------|---------|
| **产线操作员** | 生产线旁实时检测 | 快速判定 pass/fail、一键操作、清晰指示 | 大按钮、高对比度、最少步骤 |
| **质检工程师** | 结果复核、数据分析 | 查看详情、导出报告、统计分析 | 表格视图、图表展示、筛选功能 |
| **产线主管** | 生产线监控、良率追踪 | 实时良率看板、班次统计、趋势分析 | 大屏看板、仪表盘、报警提醒 |

### 1.2 典型工业检测流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  PCB上料    │───▶│ 自动采集    │───▶│ AI缺陷检测  │───▶│ 质量判定   │───▶│ 分拣下料    │
│             │    │ (相机/扫描) │    │  (≤100ms)  │    │ (Pass/Fail) │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                                                   │
       ▼                                                   ▼
┌─────────────┐                                  ┌─────────────┐
│ 记录上料时间 │                                  │ 报警器信号  │
│ 批次/型号信息│                                  │ (发现缺陷)  │
└─────────────┘                                  └─────────────┘
```

### 1.3 关键工业需求提炼

基于真实 AOI 设备调研，提炼以下核心需求：

| 需求类别 | 具体要求 | 优先级 |
|----------|----------|--------|
| **快速判定** | 单片检测 ≤150ms（含成像），快速给出 pass/fail | P0 |
| **声光报警** | 发现缺陷时触发声光报警，提醒操作员注意 | P0 |
| **防错机制** | 防止误判、漏判，关键缺陷（短路/开路）必须检出 | P0 |
| **数据追溯** | 记录每块 PCB 的检测时间、结果、图像，支撑质量追溯 | P1 |
| **连续运行** | 7×24 小时连续运行，稳定可靠 | P1 |
| **工位管理** | 支持多工位、多班次配置，适应轮班生产 | P1 |
| **统计看板** | 实时良率、缺陷分布、产能统计 | P2 |
| **报表导出** | 支持班次报表、日报表导出，满足质量体系要求 | P2 |

## 二、功能模块规划

### 2.1 运行模式设计

系统支持两种检测模式，通过现有页签切换：

| 现有代码 | 重命名 | 界面特点 | 适用场景 |
|----------|--------|----------|----------|
| ManualDetectionPanel | **手动检测** | 导入图片→检测→查看详情 | 研发/复核/抽检/文件检测 |
| RealtimeDetectionPanel | **实时检测** | 相机实时预览+Pass/Fail大字 | 生产线批量检测 |

**切换方式**：直接使用现有的 QTabWidget 页签切换，无需额外开发。

### 2.2 核心功能模块

#### 2.2.1 图像采集模块（手动检测）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 文件导入 | 支持拖拽导入、文件夹批量导入、JPG/PNG/BMP/TIFF | P0 |
| 图像预览 | 缩略图列表、放大/缩小、旋转 | P1 |
| 批量处理 | 多图片批量排队检测 | P1 |

#### 2.2.2 AI 检测模块（双模式共用）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 缺陷检测 | 6 类标准缺陷（短路/开路/缺孔/毛刺/鼠咬/多余铜皮） | P0 |
| 快速模式 | 置信度阈值预设 0.5，检测速度 ≤100ms（GPU） | P0 |
| 质量判定 | 根据缺陷类型和数量，自动判定 Pass/Fail（实时检测模式） | P0 |
| 缺陷分级 | 关键缺陷（短路/开路）→ 不良品；一般缺陷 → 依配置判定 | P0 |

**缺陷分级与判定规则**（可配置）：

```
判定逻辑：
├── 发现任意关键缺陷（短路/开路）→ FAIL
├── 发现任意多数量一般缺陷 → FAIL
├── 发现一般缺陷数量 ≤ 允收数量 → PASS（ WARN 级别）
└── 无缺陷 → PASS
```

#### 2.2.3 相机采集模块（实时检测）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 相机采集 | 支持 USB 相机 / GigE 工业相机 | P0 |
| 自动触发 | 支持外部触发（光电传感器）/ 定时触发 / 软件触发 | P0 |
| 实时预览 | 相机实时画面显示 | P0 |
| 图像预处理 | 自动调整亮度/对比度，适应不同光照条件 | P1 |

#### 2.2.4 报警与指示模块（实时检测）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 声光报警 | 发现缺陷时触发：红色指示灯 + 蜂鸣器（可配置） | P0 |
| 状态指示 | 三色指示灯：绿色（良品）/ 红色（不良）/ 黄色（等待） | P0 |
| 界面提示 | 大字显示 PASS/FAIL，结果清晰可见 | P0 |
| 报警延迟 | 支持检测到缺陷后延迟报警（避免频繁触发） | P1 |

#### 2.2.5 数据追溯模块

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 本地存储 | 所有检测数据保存在本地 SQLite 数据库 | P0 |
| 检测记录 | 记录：时间、工位、班次、图像路径、检测结果、缺陷详情 | P0 |
| 数据查询 | 支持按时间范围、工位、班次、结果类型查询 | P1 |
| 图像保存 | 可配置是否保存原始图像/标注图像（节省磁盘） | P1 |

**数据表设计**：

```sql
-- 检测记录表
detection_records (
    id              INTEGER PRIMARY KEY,
    timestamp       DATETIME NOT NULL,
    production_line VARCHAR(50),      -- 生产线
    station_name    VARCHAR(50),       -- 工位
    shift_config    VARCHAR(20),       -- 班次
    image_path      VARCHAR(500),     -- 图像路径
    result          VARCHAR(10),       -- PASS/FAIL
    defect_count    INTEGER,          -- 缺陷数量
    processing_time_ms INTEGER,       -- 检测耗时
    device_info     VARCHAR(100),     -- 设备信息
    confidence      FLOAT,             -- 置信度
    marked_image_path VARCHAR(500)    -- 标注图路径
)

-- 缺陷明细表
defect_details (
    id              INTEGER PRIMARY KEY,
    record_id       INTEGER REFERENCES detection_records(id),
    defect_type     VARCHAR(20),      -- 缺陷类型
    confidence      FLOAT,            -- 置信度
    bbox_x          INTEGER,          -- 边界框坐标
    bbox_y          INTEGER,
    bbox_width     INTEGER,
    bbox_height    INTEGER,
    severity       VARCHAR(10)        -- CRITICAL/MAJOR/MINOR
)
```

## 三、UI 交互设计

### 3.1 界面设计原则

工业场景 UI 设计遵循以下原则：

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **高可见性** | 结果在3米外可见 | 特大字号、鲜明颜色对比 |
| **最少点击** | 一次点击完成核心操作 | 一键检测、自动流转 |
| **防止误触** | 关键操作需二次确认 | 删除数据提示、模式切换确认 |
| **适应环境** | 工厂光线复杂，屏幕高亮 | 高对比度配色、可调亮度 |
| **快速反馈** | 操作立即有响应 | 即时状态提示、进度动画 |

### 3.2 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 良品/Pass | 绿色 | #22C55E |
| 不良品/Fail | 红色 | #EF4444 |
| 等待/警告 | 黄色 | #F59E0B |
| 背景（深色模式） | 深灰蓝 | #1E293B |
| 背景（浅色模式） | 浅灰 | #F8FAFC |
| 文字（深色） | 深灰 | #334155 |
| 文字（浅色） | 白色 | #FFFFFF |
| 主强调色 | 蓝色 | #3B82F6 |

### 3.3 字体与字号

| 用途 | 字体 | 字号 | 字重 |
|------|------|------|------|
| Pass/Fail 大字显示 | 微软雅黑 / 思源黑体 | 72-96px | Bold |

| 状态提示 | 微软雅黑 / 思源黑体 | 12px | Regular |

### 3.4 关键交互流程

#### 3.4.1 实时检测模式 - 连续检测流程

```
启动软件 → 默认进入 [实时检测] 页签
    │
    ├─── 相机已连接 ──▶ 显示实时预览 ──▶ [开始检测] 或 自动触发
    │
    └─── 相机未连接 ──▶ 提示连接相机 / 允许导入图片
    
    │
    ▼
检测完成 → 显示结果大屏
    │
    ├── PASS（绿色大字 + 短促提示音）
    │         └── 记录数据 → 继续等待下一片
    │
    └── FAIL（红色大字 + 报警音）
              └── 记录数据 → 等待操作员处理
              
    │
    ▼
操作员根据结果进行分拣
    │    （软件自动循环，无需点击）
    └── 可随时按 ESC 暂停
```

#### 3.4.2 手动检测模式 - 单片检测流程

```
启动软件 → 点击 [手动检测] 页签切换
    │
    ▼
[导入图像] 或 [拖拽文件]
    │
    ▼
[开始检测] 或 自动检测
    │
    ▼
查看结果（标注图 + 缺陷列表）
    │
    ├── 确认无误 → [导出报告]
    │
    └── 需要复核 → 查看原图 / 调整置信度重新检测
    
    ▼
[保存] → 自动保存到历史记录
```

## 四、功能配置项

### 4.1 工业场景设置（可配置项）

| 配置项 | 说明 | 默认值 | 范围 |
|--------|------|--------|------|
| **质量判定** ||| |
| pass_threshold | 合格阈值（缺陷数 ≤ 此值为 Pass） | 0 | 0-10 |
| fail_on_critical | 发现关键缺陷直接判定 Fail | True | True/False |
| critical_defects | 关键缺陷列表 | [short, open_circuit] | 多选 |
| alarm_on_defect | 缺陷数量达到此值时报警 | 1 | 0-10 |
| **报警设置** ||| |
| enable_sound_alarm | 启用声音报警 | True | True/False |
| alarm_sound_path | 报警声音文件路径 | "" | 文件路径 |
| alarm_delay_ms | 报警延迟（毫秒） | 500 | 0-5000 |
| **数据存储** ||| |
| save_original_image | 保存原始图像 | False | True/False |
| save_marked_image | 保存标注图像 | True | True/False |
| max_storage_days | 数据保留天数 | 90 | 7-365 |
| **生产线配置** ||| |
| production_line | 生产线名称 | default | 文本 |
| station_name | 工位名称 | default | 文本 |
| auto_shift | 启用自动班次切换 | False | True/False |

### 4.2 检测参数（专家级）

| 参数 | 说明 | 默认值 | 推荐场景 |
|------|------|--------|----------|
| confidence_threshold | 置信度阈值 | 0.5 | 通用 |

| max_defect_size | 最大缺陷尺寸（px） | 5000 | 过滤大误检 |
| enable_preprocessing | 启用图像预处理 | True | 相机实拍 |
| binarization_method | 二值化方法 | adaptive_gaussian | 不同光照 |

### 7.5 自适应机制实现

```python
# 自动检测屏幕尺寸，选择合适布局
def get_layout_mode():
    screen = QApplication.primaryScreen()
    size = screen.size()
    
    if size.width() < 1024:
        return "ultra_compact"   # 极简模式（平板）
    elif size.width() < 1280:
        return "compact"         # 紧凑模式（小屏）
    elif size.width() < 1920:
        return "standard"        # 标准模式
    else:
        return "large"           # 大屏模式
```
